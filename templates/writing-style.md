# Navigate Lab — 文章スタイルガイド

このファイルはAI編集者が記事を生成する際の「文章の質感」を規定する。
CLAUDE.md の品質基準に加えて、**このファイルのルールを必ず適用すること**。

---

## 1. 記事の「質感」の定義

Navigate Labの記事が目指す質感：

> 落ち着いた語り口で、読者の身体体験に静かに触れる。
> 説教せず、急かさず、でも確かに何かを変える。
> 読み終わった後、自分の身体を少し確かめたくなる。

この質感を作るのは、文体・禁止表現・構造の三層だ。

---

## 2. 禁止ワード・表現（全文検索で必ず確認）

### A. 禁止ワード（代替あり）

| 禁止 | 理由 | 代替表現 |
|------|------|---------|
| 整える | 既存ルール | 作る / 保つ / 立て直す / 調整する |
| ケアする | 動作が見えない | 触れる / 当てる / 動かす / 確かめる |
| 向き合う | 具体性がない | 何を、どうするかで書く |
| 大切にする | 空虚な表現 | 具体的な行動に置き換える |
| 意識する | 何をするか不明 | 注意を向ける / 感じる / 確かめる |
| 寄り添う | 使わない | — |
| ぜひ | 軽い押しつけ感 | 使わない（0回） |
| なんと | 驚きの演出 | 使わない |
| みなさん | 距離が遠い | 使わない |
| 〜ですよね | 同調の押しつけ | 使わない（1記事0回） |
| いかがでしょうか | 古い語尾 | 使わない |
| 〜してみてはいかがでしょうか | 迂回した命令 | 使わない |

### B. 禁止パターン（文末・文型）

| 禁止パターン | 理由 | 代替 |
|------------|------|------|
| 〜していきましょう | 読者に課題を押しつける | 「〜することがある」「〜から始まる」 |
| 〜ということが大切です | 説教調 | 「〜が変わる」「〜が生まれる」 |
| 〜したことはあるか / ありますか | 弱い問いかけ。バレている | 場面の描写から入る |
| 〜のではないでしょうか | 弱腰な断定 | 「〜かもしれない」か「〜だ」どちらかに決める |
| 〜かもしれませんね | 「ね」で共感を求める | 「ね」は使わない |
| 実は〜 | 1記事2回以上で安っぽくなる | 1記事1回まで |
| 〜してください | 命令 | 「試せること」セクションのみ許可 |

### C. 記号・句読点

| 禁止 | 代替 |
|------|------|
| ！（感嘆符） | 使わない。余韻は短文で作る |
| …（三点リーダー） | 使わない。間は改行と短文で作る |
| ？（疑問符）を見出しに使う | 見出しは体言か動詞活用形で終わる |

## 3. 文体ルール

### 文の長さ

- **一文の目安は40字以内**（例外：引用・列挙）
- 長い文が必要なときは「——」（2em dash）で二分する
- 段落の**最初の一文と最後の一文は短くする**（引き込みと余韻）

```
良い例：
気づいたら、使わなくなっていた。
特に決めたわけではない。ある日を境に頻度が落ち、いつしか引き出しの奥に移動していた。
捨てようとして、やめた。

悪い例：
引き出しの中には、買った当初は毎日使っていたはずなのに、気づけばいつしか使わなくなってしまったケアグッズがある。
```

### 断片文（体言止め・一語文）の使い方

- 各セクションに**1〜2回まで**使う
- 核心の一言、転換点、余韻の場面で使う

```
使える場面の例：
「道具はすでに、身体の中に入っている。」
「捨てなくていい。それでいい。」
「求めるほど、遠ざかっていく。」
```

### 「〜だ」と「〜かもしれない」のバランス

- 「〜だ。」の連続は**3回まで**（単調になる）
- 「〜かもしれない。」の連続は**2回まで**
- 確信がある事実→「だ」、読者の体験への推測→「かもしれない」で使い分ける

### ダッシュ「——」の使い方

- 転換と強調にのみ使う（1セクション2回まで）
- 前後にスペースを入れない

```
良い例：
「効いている感じ」は、変化のサインではなく——「強い刺激が来た」というサインだ。

使いすぎの例：
疲れているのは——神経系が——フル回転しているから——かもしれない。
```

## 4. フックのルール（記事冒頭）

### 始め方の原則

- **読者が経験した「具体的な場面か身体の状態」から始める**
- 統計・定義・自己紹介で始めない
- 問いかけで始めない

### 最初の3段落の構造

```
1段落目: 具体的な場面・状態の描写（3文以内）
2段落目: その状態の補足（「なぜ」「いつから」「どんな感じ」）
3段落目: 短い転換（この記事の切り口を示す一文）
```

```
良い例（moment-of-not-using-tools）：
「気づいたら、使わなくなっていた。
特に決めたわけではない。ある日を境に頻度が落ち、いつしか引き出しの奥に移動していた。買った頃の切実さは、どこかへ消えていた。
捨てようとして、やめた。」

悪い例（初稿）：
「引き出しの中に、使わなくなったケアグッズがある。
買った当初は毎日使っていた。確かに楽になる感覚があった。でも気づけば、使う頻度が減っていた。
なぜ使わなくなったのか。そして、なぜ手元に置いておくのか。」
→ 問いかけで終わっている、具体性が薄い
```

## 5. 見出しルール

- 見出しだけ読んでも記事の流れが掴める
- 「〜について」「〜の方法」「〜の重要性」は使わない
- 体言止めか動詞活用形で終わる
- 疑問符禁止

```
良い例：
「身体で起きていること」「認知の転換」「求めるほど、遠ざかる」
「首が静かになると」「道具が身体に移る」

悪い例：
「なぜ疲れるのか？」「セルフケアの方法について」「〜することの重要性」
```

## 6. 各セクションの品質基準

### 「よくある誤解」セクション

- 読者が実際に持っている誤解を1つだけ扱う
- 「〜と思いがちだ」で提示し、次の段落で静かに否定する
- 否定の仕方は断定でなく「別の見方がある」

### 「身体で起きていること」セクション

- 専門用語を1つ使う場合は括弧で初出説明
- 「だから何？」に答える一文を必ず置く
- 読者が自分の身体に照らし合わせられる具体的な感覚・場面を入れる

### 「認知の転換」セクション

- **この記事で最も重要な一文**を最後に置く
- 転換前と転換後を対比する（どちらが良いという価値判断はしない）
- 読者の「自分もそうかもしれない」を引き出す

### 「小さな身体実験」セクション

- 手順は3ステップ以内
- 「うまくいかなくてもいい」という余白を入れる
- 「なぜその実験が有効か」を1文で説明する

### 「静かな結び」セクション

- 説教で終わらない
- 「〜しましょう」「〜してください」禁止
- 記事の核心を別の角度から静かに言い直す
- 最後の一文は短く

---

## 7. 「シェアしたくなる一文」ルール

各記事に必ず1つ、以下の条件を満たす文を作る：

- **一文だけ切り取っても意味が通る**
- 逆説・再解釈・発見が含まれる
- 読者が「これ、誰かに言いたい」と思う

```
過去の記事から：
「道具はすでに、身体の中に入っている。」
「求めるほど、遠ざかっていく。」
「判断は、頭だけではなく、身体ごと行っている。」
「ひと休みという言葉は、最初からそれを知っていた。」
```

この一文は、記事の「認知の転換」か「静かな結び」に置く。

---

## 8. 読者との距離感ルール

| 使わない | 理由 | 代わりに |
|---------|------|---------|
| あなた | 近すぎる・押しつけ感 | 「人は〜」「〜する人は」「私たちは」 |
| 読者 | 外から見ている感 | 「人は〜」で書く |
| 〜してください | 命令 | 「試せること」セクションのみ |
| 我々 | 堅すぎる | 使わない |

---

## 9. まとめセクションのルール

- 箇条書きのみで終わらせない
- 箇条書きの後に1文（または箇条書きをやめて文章のみ）
- 箇条書きは4点まで
- まとめは「要約」ではなく「別の角度からの再提示」

---

## 10. 「この記事で試せること」のルール

- **1つだけ**（複数出さない）
- **太字の見出し**＋2〜3文の説明
- 手順が必要な場合は3ステップ以内
- 「うまくいかなくてもいい」「確かめるだけでいい」という余白を1文入れる
- 目的：変えること < 気づくこと
