# 記事投稿チェックリスト

1記事を「記事完成 → 画像作成 → note投稿 → SNS投稿」まで完了させる手順。

---

## PHASE 1: note投稿（15分）

### 1-1. 記事ファイルを開く

```
articles/<slug>.md
```

### 1-2. フロントマターから投稿情報を確認

```
title:    → noteの記事タイトルにコピー
subtitle: → タイトル下のサブテキスト（任意）
tags:     → noteのタグ欄に入力（カンマ区切り）
lead:     → 記事冒頭のリード文として使う
```

### 1-3. note編集画面を開く

1. [note.com](https://note.com) にログイン
2. 右上「投稿」→「テキスト」を選択

### 1-4. 本文を貼り付ける

**フロントマター（---で囲まれた部分）は貼らない。**
本文（フック冒頭の一行目）から「## 関連」の最後まで貼り付ける。

### 1-5. noteのMarkdown整形を確認

noteはMarkdownをほぼそのまま解釈する。以下を確認：

| 記号 | noteでの表示 | 対処 |
|------|------------|------|
| `# 見出し` | H1見出し | そのままでOK |
| `## 小見出し` | H2見出し | そのままでOK |
| `**太字**` | 太字 | そのままでOK |
| `——`（2emダッシュ） | そのまま表示 | そのままでOK |
| `---`（区切り線） | 水平線 | そのままでOK |

### 1-6. タイトルを入力

`title:` の内容をnoteのタイトル欄に入力。

### 1-7. タグを設定

`tags:` の内容を入力（noteは最大10個）。

### 1-8. アイキャッチ画像を設定

→ PHASE 2で作成した画像を使う（後でも設定可能）。

### 1-9. 「下書き保存」して次のPHASEへ

まだ公開しない。画像が揃ってから公開する。

---

## PHASE 2: 画像作成（30〜60分）

### 2-1. プロンプトファイルを開く

```
visuals/<slug>_prompts.md
```

5案のプロンプトが入っている。**サムネイル案1を優先して作成する。**

### 2-2. 画像生成ツールを選ぶ

| ツール | 向いている用途 | URL |
|-------|-------------|-----|
| **Canva**（推奨） | テキスト付きサムネイル、日本語対応 | canva.com |
| **Adobe Firefly** | 写真的なリアル表現 | firefly.adobe.com |
| **DALL-E（ChatGPT）** | プロンプトをそのまま入力 | chat.openai.com |
| **Midjourney** | 高品質・映画的な質感 | midjourney.com |

### 2-3. Canvaで作成する場合（最速ルート）

1. Canvaを開く → 「デザインを作成」→「Instagram投稿（縦型）」を選択
2. サイズ: **1080 × 1920px**（9:16）
3. 背景色: **オフホワイト (#F5F2ED)** か薄いグレージュ
4. フォント: Noto Serif JP / Shippori Mincho（和文）
5. プロンプトの「シーンの説明」をもとにレイアウトを作る
6. テキスト要素: 記事タイトルの核心ワードを1〜2行入れる（任意）
7. 書き出し: PNG、1080×1920px

### 2-4. AI画像生成を使う場合

プロンプトファイルの英語テキストをそのままコピーして貼り付ける。
生成後、必要に応じてCanvaで文字を追加する。

### 2-5. 必要な画像の種類

| 用途 | サイズ | 必須度 |
|-----|--------|-------|
| noteアイキャッチ | 1280×670px（横長）or 縦型 | **必須** |
| X（Twitter）投稿用 | 1200×675px | 推奨 |
| Instagramリール表紙 | 1080×1920px | 推奨 |

### 2-6. ファイル名をつけて保存

```
<slug>_thumbnail.png   → noteアイキャッチ
<slug>_x.png           → X投稿用
<slug>_reel.png        → Instagramリール表紙
```

---

## PHASE 3: note公開（5分）

### 3-1. アイキャッチ画像を設定

下書き状態の記事を開く →「アイキャッチ画像を設定」→ 作成した画像をアップロード。

### 3-2. 本文内の画像挿入（任意）

記事内に図解画像を入れたい場合：
- 挿入したい見出しの下で「+」ボタン → 画像アップロード

### 3-3. 最終確認

- [ ] タイトル入力済み
- [ ] タグ設定済み（2〜5個）
- [ ] アイキャッチ画像設定済み
- [ ] 本文内に「整える」「ケアする」などの禁止ワードがないか目視確認
- [ ] 「この記事で試せること」が末尾にある

### 3-4. 公開設定

- 公開範囲: **全体に公開**
- 価格: **無料**（シリーズ構築期間中は無料推奨）

### 3-5. 「公開する」ボタンを押す

公開されたURLをコピーしておく。

---

## PHASE 4: SNS投稿（10分）

### 4-1. SNSファイルを開く

```
social/<slug>_sns.md
```

### 4-2. X（Twitter）投稿

1. X を開く
2. `## X投稿（3種類）` の中から1つ選ぶ（**① 問いかけ型** を最初に推奨）
3. テキストをコピーして貼り付け
4. 公開したnoteのURLを末尾に追加
5. X用画像（`<slug>_x.png`）を添付
6. 投稿

残り2種類は3〜5日後に時間をずらして投稿する。

### 4-3. Instagram投稿

1. Instagramを開く
2. リール台本（`## Instagramリール台本`）をもとに動画を撮影 or スライド作成
3. 表紙画像（`<slug>_reel.png`）を設定
4. キャプション: X投稿文を参考に140字程度で書く
5. ハッシュタグを5〜10個追加
6. 投稿

### 4-4. note導入文の活用

`## note導入文` の内容は、noteのコメント欄や他プラットフォームのリンク共有時に使う。

---

## PHASE 5: GitHubのIssueをCloseする

1. GitHubのリポジトリを開く
2. Issues → 該当のIssue を開く
3. 「Close issue」をクリック
4. コメント欄に公開したnoteのURLを貼り付けてから閉じる

---

## 投稿完了の確認

- [ ] note公開済み（URLあり）
- [ ] X投稿済み（1本目）
- [ ] Instagramリール / 投稿済み
- [ ] GitHubのIssue Closed
- [ ] GitHubのPRがマージ済み

---

## よくある詰まりポイント

| 問題 | 対処 |
|------|------|
| noteで改行が多すぎる | 段落間の空行が2行以上あると余白が大きくなる。1行に減らす |
| 見出しが大きすぎる | noteの`#`はH1。記事内の見出しは`##`から使うと自然 |
| 画像が横長になってしまう | アイキャッチは1280×670pxが標準。縦型は本文内挿入向き |
| Canvaの書き出しが遅い | PNG→JPGに変更すると速くなる（note投稿なら画質十分） |
