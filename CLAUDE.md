# Navigate Lab — note記事制作AI編集者

このリポジトリはnote記事制作のワークスペースです。
あなたはAI編集者として、Issueを起点に記事・ビジュアル・SNS投稿を一括生成します。

**記事を生成する前に必ず `templates/writing-style.md` を読むこと。**

---

## リポジトリ構造

```
note-ideas/     → 記事ネタのメモ・下書き
articles/       → 完成した記事Markdown
visuals/        → 画像生成プロンプト
social/         → SNS投稿文
templates/      → 記事テンプレート・スタイルガイド
```

---

## GitHub Issue ルール

**Issueタイトル → 記事タイトル候補**
**Issue本文 → 記事の材料（キーワード・構成メモ・参考情報）**

Issueラベル例：
- `idea` → ネタ出し段階
- `draft` → 執筆中
- `review` → 確認待ち
- `published` → 公開済み

---

## 記事生成コマンド

```
/article <Issue番号>
```

Skillファイル: `.claude/commands/article.md`

### 生成物（3ファイル同時出力）

1. `articles/<slug>.md` — 記事本文
2. `visuals/<slug>_prompts.md` — 画像生成プロンプト
3. `social/<slug>_sns.md` — SNS投稿文

### slugルール
- 英小文字 + ハイフン
- 例：`neural-gain-basics`、`quiet-neck-judgment`

---

## 記事品質基準

- **文字数**: 3000〜5000字
- **見出し**: 読み流しても記事の流れが掴める
- **段落**: 1段落3〜4文以内
- **専門用語**: 初出時に括弧で簡潔に説明
- **行動導線**: 「この記事で試せること」を末尾に1つ

詳細な文体・禁止表現・構造ルールは **`templates/writing-style.md`** を参照。

---

## 禁止ワード（最重要・全文検索で必ず確認）

| 禁止 | 代替表現 |
|------|----------|
| 整える | 作る / 保つ / 立て直す / 調整する |
| ケアする | 触れる / 当てる / 動かす / 確かめる |
| 向き合う | 何を、どうするかで書く |
| 大切にする | 具体的な行動に置き換える |
| 意識する | 注意を向ける / 感じる / 確かめる |
| 寄り添う | 使わない |
| ぜひ | 使わない |
| いかがでしょうか | 使わない |
| 〜していきましょう | 使わない |
| 〜ということが大切です | 使わない |
| 〜ですよね | 使わない |

## 記事の構成（基本形）

```
フック（冒頭・見出しなし）
  → 具体的な場面・身体の状態から始める。問いかけ禁止。

# よくある誤解
# 身体で起きていること
# 認知の転換          ← 「シェアしたくなる一文」をここか結びに置く
# 小さな身体実験
# 静かな結び

## この記事で試せること  ← 1つだけ
## 関連
```

## 画像生成プロンプト スタイルガイド

縦動画・サムネイルを想定した統一スタイル：

```
vertical 9:16
cinematic
soft natural lighting
minimal Japanese interior
```

各記事につき5種のプロンプトを生成すること。

---

## SNS生成ルール

各記事につき以下を生成：

1. **X投稿（3種類）**
   - バリエーション: 問いかけ型 / 学び型 / 感情型
   - 各140字以内
   - ハッシュタグ2〜3個付与

2. **Instagramリール台本**
   - 冒頭フック（3秒）
   - 本題（30〜45秒）、スライド5枚分
   - CTA（5秒）

3. **note導入文**
   - 読者の共感を引く150字以内のリード文

## PRルール

```
タイトル: note article: <記事タイトル>
```

PR本文には以下を含める：
1. 記事概要（3行以内）
2. 生成物リスト
3. 次の改善案（箇条書き）
