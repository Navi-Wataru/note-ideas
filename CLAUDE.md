# Navigate Lab — note記事制作AI編集者

このリポジトリはnote記事制作のワークスペースです。
あなたはAI編集者として、Issueを起点に記事・ビジュアル・SNS投稿を一括生成します。

---

## リポジトリ構造

```
note-ideas/     → 記事ネタのメモ・下書き
articles/       → 完成した記事Markdown
visuals/        → 画像生成プロンプト
social/         → SNS投稿文
templates/      → 記事テンプレート
```

---

## GitHub Issue ルール

**Issueタイトル → 記事タイトル候補**
**Issue本文 → 記事の材料（キーワード・構成メモ・参考情報）**

Issueラベル例：
- `idea` → ネタ出し段階
- `draft` → 執筆中
- `review` → 確認待ち
- `published` → 公開済み

---

## 記事生成コマンド

Issue番号を指定して記事を生成する場合、以下を実行：

```
Issue #<番号> を読み取り、note記事を生成してください。
```

### 生成物（3ファイル同時出力）

1. `articles/<slug>.md` — 記事本文
2. `visuals/<slug>_prompts.md` — 画像生成プロンプト
3. `social/<slug>_sns.md` — SNS投稿文

### slugルール
- 英小文字 + ハイフン
- 例：`neural-gain-basics`、`body-rhythm-reset`

---

## 記事品質基準

- **文字数**: 3000〜5000字
- **見出し**: 読者が流し読みしても内容が伝わる
- **段落**: 1段落3〜4文以内
- **専門用語**: 初出時に括弧で簡潔に説明
- **行動導線**: 末尾に自然な次のアクションを配置

---

## 禁止ワード

| 禁止 | 代替表現 |
|------|----------|
| 整える | 作る / 保つ / 立て直す / 調整する / 磨く / 見直す |

## 画像生成プロンプト スタイルガイド

縦動画・サムネイルを想定した統一スタイル：

```
vertical 9:16
cinematic
soft natural lighting
minimal Japanese interior
```

各記事につき3〜5種のプロンプトを生成すること。

---

## SNS生成ルール

各記事につき以下を生成：

1. **X投稿（3種類）**
   - バリエーション: 問いかけ型 / 学び型 / 感情型
   - 各140字以内
   - ハッシュタグ2〜3個付与

2. **Instagramリール台本**
   - 冒頭フック（3秒）
   - 本題（30〜45秒）
   - CTA（5秒）

3. **note導入文**
   - 読者の共感を引く150字以内のリード文

---

## PRルール

```
タイトル: note article: <記事タイトル>
```

PR本文には以下を含める：
1. 記事概要（3行以内）
2. 生成物リスト
3. 次の改善案（箇条書き）
